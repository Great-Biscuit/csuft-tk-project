## 协作文档
### Commit 规范
- feat：增加了新功能
- fix：修补 bug
- docs：文档
- style：格式变化，不影响代码运行的变动
- refactor：重构代码（即不是新增功能，也不是修改bug的代码变动）
- test：增加测试
- chore：构建过程或辅助工具的变动

在每次 commit 时，其 message 必须使用如上格式开头！

例如：
- 命令行方式： `git commit -m "feat: 新增登录页面"` 
- IDEA：在提交消息中使用上述命名格式
